## 1. 产品概述
店声通——一款面向本地生活商家的移动端口碑监测工具，帮助店长每天用十分钟掌握贴吧、城市论坛中关于自家门店的真实讨论。解决"顾客背后怎么说你却不知道"的痛点，让问题早发现、回复有策略、整改有方向。
- 目标用户：餐饮、美业、教培等本地生活门店的店长/老板
- 核心价值：将分散在社区论坛的零散口碑转化为可执行的每日行动清单

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 店长 | 首次输入门店信息即可使用 | 查看口碑动态、归类问题、管理回复策略 |

### 2.2 功能模块
1. **引导页**：首次使用输入门店名、商圈名、老板常用称呼、服务项目
2. **今日提到我**：按时间流展示新帖子，标签提示吐槽类型，可展开查看原帖语境与回复观点
3. **问题归类**：近七天吐槽合并成可执行清单，如"周末等位被反复提到"
4. **回复提醒**：区分"适合公开解释"和"只需内部整改"的帖子，避免盲目回帖引发二次争议

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 引导页 | 门店信息表单 | 输入门店名、商圈名、老板称呼、服务项目（多选标签），完成后进入首页 |
| 今日提到我 | 时间流列表 | 按时间倒序展示提及门店的帖子，每条显示来源、时间、摘要、标签（排队久/价格贵/态度好/环境差等） |
| 今日提到我 | 帖子详情 | 展示原帖全文/截图、主要回复观点摘要、是否有人附图标记 |
| 问题归类 | 问题聚合卡片 | 按类别聚合近七天投诉，如"等位问题(5次)"，点击展开相关帖子列表 |
| 问题归类 | 行动建议 | 每个问题类别下给出简短可执行建议，如"建议增设周末预约通道" |
| 回复提醒 | 公开解释列表 | 适合公开回复的帖子，标注推荐回复策略（道歉/解释/邀请体验） |
| 回复提醒 | 内部整改列表 | 不适合公开回复但需内部关注的问题，标注整改方向 |
| 回复提醒 | 回复草稿 | 为公开解释类帖子提供回复草稿模板，可编辑后一键复制 |

## 3. 核心流程

店长首次打开应用 → 填写门店信息 → 进入「今日提到我」首页浏览新帖 → 点击帖子查看详情与回复观点 → 切换到「问题归类」查看近七天问题汇总与行动建议 → 切换到「回复提醒」确定哪些帖子公开回复、哪些内部整改 → 对公开回复帖使用草稿模板组织回复

## 4. 用户界面设计

### 4.1 设计风格
- **主题色调**：暖色系——以琥珀橙(#E8913A)为主色、暖灰(#F5F0EB)为底色，营造"每天清晨看一眼"的温暖日常感
- **辅助色**：标签系统使用语义色——红色(排队久/环境差)、黄色(价格贵)、绿色(态度好/推荐)
- **按钮风格**：圆角胶囊按钮，柔和阴影，点击有轻微缩放反馈
- **字体**：标题使用 Noto Serif SC（衬线感带来"日报"质感），正文使用 Noto Sans SC
- **布局风格**：移动端全屏卡片流，底部Tab导航，内容区固定高度内滚动
- **图标风格**：线性图标（lucide-react），2px描边，与文字等宽对齐

### 4.2 页面设计概述
| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 引导页 | 门店信息表单 | 全屏暖色渐变背景、居中卡片式表单、输入框带图标前缀、服务项目标签多选、底部CTA按钮 |
| 今日提到我 | 时间流列表 | 顶部日期选择器、卡片式帖子列表、左侧时间轴竖线、标签胶囊、来源图标、附图角标 |
| 今日提到我 | 帖子详情 | 顶部返回导航、原帖引用区块（灰色背景）、回复观点摘要卡片、附图缩略图网格、底部操作栏 |
| 问题归类 | 问题聚合卡片 | 分类标题+频次徽章、可展开手风琴、展开后显示相关帖子列表、每类底部行动建议条 |
| 回复提醒 | 双栏列表 | 顶部Tab切换"公开解释/内部整改"、卡片列表含推荐策略标签、公开帖显示草稿按钮、内部整改帖显示整改方向标签 |

### 4.3 响应式设计
- 移动优先设计，最大宽度 430px 居中展示
- 底部固定Tab导航栏，适配安全区域
- 触摸优化：卡片点击区域 ≥ 44px，按钮 ≥ 48px
- 内容区固定视口高度内滚动，避免整页滚动

### 4.4 3D场景指导
- 不适用，本应用为2D信息展示型应用
